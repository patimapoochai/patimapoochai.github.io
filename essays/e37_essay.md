Websites are built using UI frameworks as the industry standards. Go on a web framework’s website, you would see many logos of companies that use the framework. Big companies even publish their own UI and front-end frameworks, like Facebook’s React. 
In this industry, keeping up with the industry standards is important for successful employment and project development. 
Your clients, employers, and companies would expect you to work in the same environment, procedure, and pipelines as them, so understanding the tools they use is important.
Creating your own startups, even if you could define your own frameworks or even not using any frameworks at all, will benefit greatly from following the industry standard. Frameworks that big companies used are proven to be beneficial towards your project, for their success likely depend upon the framework being beneficial towards the company. In addition to that, if you want to replicate or reach the successes that big companies achieve, using their tools would allow you to potentially reach the same success.
These UI frameworks are tedious, sure, but the usage of these frameworks in the technology industry requires you to learn them. Even if UI frameworks are difficult and time-intensive, learning them could help you in your web-development career.


Still, non-conformity and thinking outside of the box is good for innovation in this industry. If tracing the success of big companies, perhaps looking at the reason why big companies use UI frameworks would be more convincing.
Working with frameworks save time and it does stuff for you better than you could yourself
You don’t have to build complex elements from scratch. UI frameworks like Semantic UI already have collections of common components premade for you. The framework streamlines the development process, allowing you to focus on designing the site, rather than wrestling with the CSS to complete a single element.
UI frameworks are being developed by multiple professional coders. The source code of Semantic UI is openly hosted on Github, allowing hundreds of contributors to fix bugs and refine each component. Creating your own UI with raw HTML and CSS has the benefit of better understanding and better control, but you probably can’t compare to the reliability and headache by letting someone else build each component for you. UI frameworks also have the benefit of having a large support community. An open source project like Semantic UI hosts discussion forums for maintainers to development discussion. This communication platform also dedicates a space for anyone to post questions, report bugs, and request features for the frameworks, allowing users to ask questions and request tech support. The large pool of people to request assistance in styling your project doesn’t compare to a single person trying to code everything by themself. Due to the larger amount of maintainers, using a UI framework allows your project to look like it has been collaborated by a large team of developers.
