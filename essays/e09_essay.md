Javascript seems like the most uncoordinated language when I first used it during a hackathon. I chose to work on the front-end for the Scholar System project (check it out on my projects page!), and since our front-end lead was very familiar with it, I had to learn the language. Mind you, I learn programming through typed, object-oriented languages, like Java and C/C++. My paradigm about programming has always been: "draw a boundary box that your program will always operate in." Javascript doesn't have a boundary since I could just "conjure" up new variables without declaring its type. One particular instance of this observation is when I was working through [freeCodeCamp](https://www.freecodecamp.org/), where one of the lessons introduced the language's ability to any properties to an object. To me, this goes against my basic understanding of programming. It took a while for my paradigm to shift toward: "you can add a random variable to an object's member variable."

While this characteristic of Javascript seems to make the language clumsy at first, I could see its usefulness after working through an assignment for the ICS314 college class. In web development, being able to adaptively and quickly display a user's properties is very useful. Each user could have a different amount of values that need to be displayed, and these values won't necessarily be definable beforehand like typed languages. For example, my resume page won't necessarily have a top-selling-novels or theater-plays-casted section, but others might. Instead of deleting a class definition or writing an exhaustive amount of code to be able to adaptively add and remove sections, these functions are built into the language. Just like Jackie Chan using Drunken Boxing, Javascript feels uncoordinated, but concise at the same time.

Though I like this property of Javascript now, I wouldn't be able to learn about it without the ICS314 course athletic software engineering pedagogy. The overall idea is to subject students to a lot of coding practices, then place them in time-sensitive tests called WODs (Work Out of the Day). So far, the WODs are easy because they're only introducing students to Javascript. However, I can say for sure that the WODs would get harder and more stressful as time goes on. There is an upside to this, however, as this learning style forces students to repeatedly practice the material. Even in the introductory phase, where the material could get boring, I still reviewed the material. Knowing that I have to perform in a high-stress environment, I am more perceptive towards small details like Javascript.
