[Rainbow Retaik](https://renigmaflea.github.io/) is the final project I and my teammates submitted for a software development class at the and of the spring 2020 semester. The prompt for the app is to create a shopping platform for the students dorming at the University of Hawaii dorms. We based the on popular sites like [Craiglist](https://www.craigslist.org) or [Facebooks's Marketplace](https://www.facebook.com/marketplace/), where there's no explicit payment system. Rather, each users would negotiate with each other for the price of an item. The app was built using [React](https://reactjs.org/), [Meteor](https://www.meteor.com/#!), [MongoDB](https://www.mongodb.com/), and [Semantic UI](https://semantic-ui.com/). The source code is hosted on [Github](https://github.com/renigmaflea/renigmaflea).

While the the choice of frontend and backend tools were restricted by the professor, this project was a very good introduction to website technology stack. React is a very versatile and easy to learn frontend tech, and the franework meshes with Semantic UI very nicely. Meteor and MongoDB is quite intuitive after I understood the documentation. The ease and simplicity of the stack allowed me to work on both the frontend and backend of the app. I was able to create the user-to-user chat system and admin reporting system, with related server code and website code, entirely from scratch. Since both front and back-end uses javascript, making the user access control was also easily accomplished.

This was my second project with Github, but it is the first project that utilizes an Agile-like project management. There were several problems with this management style, since we weren't experienced in the process. Three milestones were established, with 10 to 20 tasks created for each milestone. The process boils down to holding a meeting every three days to discuss the state of the project while we work on each task separately. Some mistakes in planning and communication were made, but we were able to come together and deliver a functional product.

Through making this application, I'm very excited to learn more about creating web applications. The process and tools that we've learned is very close to the current industry standard, so it's possible to use the same frameworks and tools to make the next Facebook or Ebay. I've taken one step closer towards my software engineering career.
